Deployment Pipeline
Step-by-step from trained model to live production system
Step 01
Model Serialisation
Trained ensemble saved via joblib.dump(). Files: xgb_v2.pkl, lgb_v2.pkl, lstm_v2.h5, ensemble_v2.pkl. Total model size: 48 MB.
Step 02
REST API with FastAPI
Wrapped in a FastAPI application with POST /predict endpoint. Accepts player feature vector, returns predicted transfer value in EUR from all 4 models.
Step 03
Dockerisation
Containerised with Docker. Image size: 2.1 GB. Includes Python 3.11, model weights, all dependencies. Runs identically across dev, staging, and production.
Step 04
Cloud Hosting (AWS EC2)
Deployed on AWS EC2 t3.medium behind an Application Load Balancer. Auto-scaling group handles traffic spikes. 99.9% uptime SLA.
Step 05
CI/CD Pipeline
GitHub Actions triggers retrain + redeploy when new season data is pushed. Model validation gate: if R² drops below 0.90 on holdout, deployment is blocked automatically.
Step 06
This Dashboard (Front-end)
Interactive visualisation layer built in HTML/JS. Deployed as a static site on AWS S3 + CloudFront. Calls the prediction API in real time to serve scouts and analysts.

Key Findings
Ensemble Best Model
Stacking LSTM + XGBoost + LightGBM achieves R²=0.931, outperforming each model individually by combining temporal, boosting, and meta-learner signals.
Sentiment Predicts Value
VADER compound + social buzz contribute 19.2% of XGBoost feature gain, confirming social sentiment is a statistically significant predictor of transfer value shifts.
XGBoost Tuning +5.5%
Grid search over 64 combinations improved R² from 0.872 (default) to 0.920. learning_rate was the most sensitive parameter at 28% of gain.
Position Error Gap
Forwards show highest RMSE (€2.1M) due to volatile scoring. Defenders most predictable (€1.4M). Position-specific models recommended for future work.
